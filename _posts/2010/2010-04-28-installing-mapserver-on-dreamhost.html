--- layout: post title: Installing Mapserver on Dreamhost tags: - technology status: draft type: post published: false meta: _edit_last: "1" --- I've used dreamhost for a number of years and in general I've been pretty happy. They provide a decent linux host with just about everything you need to do web stuff. I'd prefer postgres to mysql, and it'd be nice to have tomcat, but that sort of thing is available with their virtual server. If I can't have geoserver on dreamhost, I'd at least like mapserver. Without root access, it's possible to compile and install it from source. I seem to remember a perrygeo post on that subject, but I can't seem to find it right now. It is a fairly straightforward process, following the steps from the verbose linux install. You have to make a few changes to install the newly compiled software to a location other than root. I've chosen /home/eknuth/install for the location of the binaries. When the instructions call for you to run ./configure, add --with-prefix=/home/eknuth/install to the statement. Install Prerequisites Download the latest stable gdal source from http://trac.osgeo.org/gdal/wiki/DownloadSource and put it in a built folder. I used /home/eknuth/build. Uncompress it and cd to the gdal folder. Configure: ./configure \ --with-png \ --with-libtiff \ --with-jpeg \ --with-gif \ --with-geos \ --build=i686-pc-linux-gnu \ --host=i686-pc-linux-gnu \ --prefix=/home/eknuth/install Build and Install: make make install
